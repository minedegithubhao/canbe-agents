项目更新后，更新 WSL Docker 里的服务，本质就是：重新 build 镜像 + 重启对应容器。

我建议默认用这个流程：

``` cmd
wsl sh -lc "cd /mnt/d/AAA_tools && docker compose -p aaa_tools -f docker-compose.yaml -f docker-compose.agent.yaml up -d --build canbe-agents canbe-blog-server canbe-blog-web"
```

如果你只改了某一个项目，可以只更新对应服务：

# 只更新 canbe_agents

``` cmd
wsl sh -lc "cd /mnt/d/AAA_tools && docker compose -p aaa_tools -f docker-compose.yaml -f docker-compose.agent.yaml up -d --build canbe-agents"
```

# 只更新 canbe_blog_server

``` cmd
wsl sh -lc "cd /mnt/d/AAA_tools && docker compose -p aaa_tools -f docker-compose.yaml -f docker-compose.agent.yaml up -d --build canbe-blog-server"
```

# 只更新 canbe_blog_web

```cmd
wsl sh -lc "cd /mnt/d/AAA_tools && docker compose -p aaa_tools -f docker-compose.yaml -f docker-compose.agent.yaml up -d --build canbe-blog-web"
```

更新后建议验证：

``` cmd
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8801/health
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8800/actuator/health
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:3000
```

我的建议是：开发阶段 Windows 本地调试，阶段性改完后用上面的 Compose 命令刷新 WSL Docker 环境。这样既保留 IDE 调试效率，又能保证最终运行方式接近部署环境。
