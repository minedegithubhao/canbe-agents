# RAG Lab V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first usable version of a RAG experiment lab that can version datasets and pipelines, generate eval sets, run experiments, compare runs, and inspect case-level traces.

**Architecture:** Keep the current repo as the execution base, but pivot the product center from `/faq/chat` to `Experiment`. Use FastAPI as the control plane, MySQL for experiment metadata, artifact files for large traces and reports, and a background worker for dataset build, experiment run, and comparison jobs. Preserve reusable retrieval logic from the existing project, but reorganize it behind experiment-oriented domain services.

**Tech Stack:** FastAPI, Pydantic, SQLAlchemy or SQLModel, MySQL, background worker queue, local artifact storage, existing retrieval/rerank/provider integrations, pytest

---

## File Structure

### New or reworked units

- `app/domain/`
  - Responsibility: core experiment entities and enums
- `app/repositories/mysql/`
  - Responsibility: MySQL persistence for datasets, pipelines, eval sets, runs, comparisons
- `app/services/dataset_service.py`
  - Responsibility: dataset version ingestion and indexing orchestration
- `app/services/pipeline_service.py`
  - Responsibility: pipeline and pipeline version management
- `app/services/eval_service.py`
  - Responsibility: eval set generation and case maintenance
- `app/services/experiment_service.py`
  - Responsibility: run creation, execution orchestration, result aggregation
- `app/services/comparison_service.py`
  - Responsibility: run comparison and verdict generation
- `app/services/artifact_service.py`
  - Responsibility: trace and report file persistence
- `app/api/rag_lab_*.py`
  - Responsibility: control plane APIs for datasets, pipelines, eval sets, runs, comparisons
- `tests/rag_lab/`
  - Responsibility: unit and integration tests for each new module

  - Register new experiment lab routers and startup services
- `app/settings.py`
  - Add MySQL, artifact storage, and worker config
- `app/services/retrieval_service.py`
  - Extract reusable runtime-facing methods without FAQ-specific assumptions
- `app/services/chat_service.py`

### Docs
